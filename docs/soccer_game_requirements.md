# サッカータクティクスゲーム：要件定義書

**プロジェクト名：** Soccer Tactics Arena（仮）  
**作成日：** 2026年5月27日  
**バージョン：** 1.0  
**対象者：** PM、エンジニア、デザイナー

---

## 📋 Executive Summary

本アプリは、**ターンベース形式の戦術シミュレーションサッカーゲーム**です。プレイヤーは監督として**フォーメーション選択と戦術設定**を行い、その後の試合展開は自動解決されます。短時間プレイ（5分以下）を想定した、サッカーファンからカジュアルゲーマーまで幅広い層を対象とした作品です。

---

## 🎮 1. ゲームコンセプト

### 1.1 ゲームジャンル・タイプ
- **ジャンル：** タクティクスシミュレーション
- **プレイスタイル：** ターンベース戦術ゲーム（チェスのような思考ゲーム）
- **視点：** 監督視点（俯瞰視点）
- **対戦形式：** PvP（人間同士）と PvE（AI対戦）の両方に対応

### 1.2 基本ゲームループ
```
1. ロビー画面 → 対戦相手を選択/マッチング
2. チーム編成画面 → フォーメーション・戦術を選択
3. プリマッチ → 両プレイヤーが戦術を設定（同時または順番）
4. 試合シミュレーション → AI判定で試合展開が自動解決
5. リザルト画面 → 試合結果表示・スタッツ確認
6. リロビー → 次の対戦へ
```

### 1.3 プレイヤーの達成感ポイント
- ✅ **戦術の成功** → 自分の采配が成功し、相手を倒す快感
- ✅ **短いサイクル** → 5分以内に完結するため、達成感が頻繁に得られる
- ✅ **思考ゲーム要素** → チェスのような戦術判断が評価される

## 🎯 2. ターゲットユーザー（複数ペルソナ）

### ペルソナA：サッカーファン（全年代）
- **特性：** サッカーに詳しく、リアルな戦術に興味がある
- **プレイ動機：** 実際のサッカー知識を活かして遊びたい
- **求める要素：** リアルなフォーメーション、実名選手など

### ペルソナB：カジュアルゲーマー
- **特性：** ゲーム好きだが、サッカーにはそこまで詳しくない
- **プレイ動機：** シンプルな戦術ゲームとして遊びたい
- **求める要素：** ルールが分かりやすく、短時間で楽しめる

### ペルソナC：隙間時間ゲーマー
- **特性：** 通勤・待ち時間など、短い時間でプレイしたい
- **プレイ動機：** 手軽に遊べる対戦ゲーム
- **求める要素：** サクサク動く、待ち時間が短い

## ⏱️ 3. ゲーム時間軸

### 3.1 プレイ時間
- **1試合の所要時間：** 3〜5分
  - チーム選択：30秒
  - 戦術設定：30秒
  - 試合シミュレーション：2分
  - リザルト確認：30秒

### 3.2 ゲーム進行スタイル
- **短時間単発型** ← 選択済み
- 1試合ごとに完結し、リーグやシーズンシステムはなし
- 何度でも対戦を繰り返すことが可能

## 🏆 4. ゲームプレイシステム

### 4.1 ゲーム開始フロー

#### ステップ1：対戦相手の選択
- **PvP：** マッチメイキングシステムで対戦相手を探す
  - ランダムマッチング
  - レート別マッチング（将来実装）
- **PvE：** CPU相手を選択
  - 難易度選択（イージー・ノーマル・ハード）

#### ステップ2：チーム編成 & 戦術設定

**フォーメーション選択（複数プリセット）**
- 4-3-3（バランス型）
- 4-2-4（攻撃型）
- 5-3-2（守備型）
- 3-5-2（サイド攻撃型）
- その他複数

**戦術プリセット選択**
- パス主導型
- ロングボール主体
- サイド攻撃重視
- セットプレー重視
- など複数パターン

### 4.2 対戦フロー

#### フェーズ1：準備フェーズ（同時または順番に実施）
- 両プレイヤー/CPU がフォーメーション＆戦術を選択
- PvP時は相手が戦術を見えない状態で設定（同時決定）

#### フェーズ2：試合シミュレーション
- システムが両チームの戦術に基づいて試合を自動実行
- シミュレーション中はプレイヤーは操作不可
- 実況アニメーションまたはテキストで展開を表示

#### フェーズ3：リザルト表示
- 最終スコア
- 主要統計（シュート数、パス成功率、ポゼッション率など）
- MVP選手
- 勝敗判定

### 4.3 戦術シミュレーションロジック（AI判定の基本構造）

```
戦術相性度 = 攻撃型式の有効性 + 相手戦術への相性 - ランダム要素

例：
- 4-3-3（バランス）vs 4-2-4（攻撃）
  → 相手は攻撃的だが、バランス型で対抗
  → シューティング精度、クロスの質などで勝敗が決定
  
- ロングボール vs パス主導
  → ロングボールは相手のプレスをかわしやすい
  → ただしターンオーバーが増える可能性も
```

**重要：** 完全なランダムではなく、戦術相性に基づいた確率的結果

## 📊 9. コアゲームメカニクス詳細

### 9.1 フォーメーション相性表

```
          vs  4-3-3  4-2-4  5-3-2  3-5-2
4-3-3        50%    45%    55%    50%
4-2-4        55%    50%    45%    60%
5-3-2        45%    55%    50%    48%
3-5-2        50%    40%    52%    50%
```
**相性%：** その行のフォーメーションが列のフォーメーションに対して勝つ基本確率

### 9.2 戦術相性の詳細

**攻撃型戦術：**
- パス主導（ショートパス重視）→ ボール保持率UP、シュート精度UP、ターンオーバーリスク低
- ロングボール（カウンター主体）→ シュート精度DOWN、カウンター成功率UP、ターンオーバーリスク高
- サイド攻撃 → クロス精度UP、センターFW依存度UP

**防御型戦術：**
- 堅い守備 → 失点リスク低、攻撃機会少
- ハイプレス → 奪いどき多、フィジカル消費大
- セットプレー重視 → CK/FK得点率UP

### 9.3 スタッツシステム

**試合後に表示する統計情報：**
```
基本統計
├─ シュート数（枠内外）
├─ パス本数 / パス成功率
├─ ポゼッション率
├─ タックル数
├─ インターセプト数
└─ クロス本数 / 成功率

得点関連
├─ 得点源（誰が決めたか）
├─ アシスト
├─ シュート精度
└─ ビッグチャンス

オフェンス/ディフェンス
├─ CK数
├─ FK数
├─ イエローカード
├─ レッドカード
└─ PK得失点
```

## 💡 14. リスク・課題・検討事項

### 課題1：ゲームバランス
- **リスク：** 特定のフォーメーション/戦術が強すぎる可能性
- **対策：** ベータテスト、継続的な相性調整

### 課題2：マッチング
- **リスク：** ユーザー数が少ないと待ち時間が長くなる
- **対策：** MVP時点ではCPU主体、段階的なPvP導入

### 課題3：リテンション
- **リスク：** 単発対戦型のため飽きやすい可能性
- **対策：** チャレンジシステム、限定イベント検討

### 課題4：ライセンス
- **リスク：** 実名選手・クラブ使用時のライセンス費用
- **対策：** MVP時は汎用選手名で実装、後にライセンス交渉

### 課題5：GDPR / プライバシー
- **リスク：** EU圏でのコンプライアンス
- **対策：** プライバシーポリシー策定、適切なデータ管理
